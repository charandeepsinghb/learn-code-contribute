Basic Git Commands and Instructions


1. Install Git

If Git is not already installed on your system, you can download and install it from the official website.


2. Configure Git

After installation, configure Git with your name and email address. This information will be associated with your commits.

    git config --global user.name "Your Name"
    git config --global user.email "youremail@example.com"

3. Create a Git Repository

You can either create a new Git repository for your project or clone an existing one.

To create a new repository, navigate to your project's directory and run:

    git init

To clone an existing repository, use the following command, replacing <repository_url> with the URL of the remote repository:

    git clone <repository_url>

4. Working Directory, Staging Area, and Repository

In Git, you have three main areas:

  • Working Directory: This is where you make changes to your files.
  • Staging Area (Index): Files you want to commit are staged here.
  • Repository (HEAD): This is where committed changes are stored.

5. Basic Git Workflow

Here's the basic Git workflow:

  1. Make changes to your files in the working directory.
  2. Use the git add command to stage the changes for the next commit:
    3.     git add <file_name>
  3. Commit your changes to the repository with a meaningful message:
    4.     git commit -m "Your commit message here"

6. Viewing the Repository

Use git status to see the current status of your repository, including tracked and untracked files.

Use git log to view a history of commits.


7. Working with Branches

Branches allow you to work on different features or bug fixes independently.

To create a new branch:

    git checkout -b <branch_name>

To switch between branches:

    git checkout <branch_name>

To merge branches:

    git merge <branch_name>

To delete a branch (after it has been merged):

    git branch -d <branch_name>

8. Remote Repositories

Git allows you to collaborate with others by pushing and pulling changes from remote repositories (e.g., on GitHub, GitLab, Bitbucket).

To add a remote repository:

    git remote add origin <repository_url>

To push your changes to a remote repository:

    git push -u origin <branch_name>

To pull changes from a remote repository:

    git pull origin <branch_name>

9. Handling Merge Conflicts

If two people make conflicting changes, Git will report a merge conflict. You'll need to manually resolve these conflicts in the affected files, then commit the resolved files.


10. Undoing Changes

To discard changes in the working directory:

    git checkout -- <file_name>

To unstage a file (remove from the staging area):

    git reset HEAD <file_name>

To undo the last commit (be cautious with this as it reverts the entire commit):

    git reset --hard HEAD^
